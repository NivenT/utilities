1 # TypeMap
2 
3 This provides a map from types to values of the type. In particular, this map let's you store one value of each type (including custom types and references).
4 
5 ## Example Usage
6 ```C++
7 #include <TypeMap.h>
8 
9 int main() {
10  int b = 8;
11 
12  TypeMap map;
13  map.insert<int>(5);
14  map.insert<char>('a');
15  map.insert<int&>(b);
16 
17  assert(map.find<int>() == 5);
18  assert(map.find<int&>() == 8);
19  assert(map.find<char>() == 'a');
20 
21  map.get<int>()++;
22  assert(map.get<int>() == 6);
23 
24  // Inserts a float into the map
25  map.get<float>() = 3.5;
26  assert(map.find<float>() == 3.5);
27 
28  // Crashes at runtime because no double has been asserted
29  map.find<double>() = 4;
30 }
31 ```

33 ## Details
34 * Internally data is stored in a `std::unordered_map<utils::TypeInfo, void*>`.
35 * The TypeMap calls all necessary destructors when elements are removed just like a `std::vector` or `std::map` would even though it stores `void*`.
36 * You can access the raw `void*` pointers if you really want to via `TypeMap::find(TypeInfo)`. If you do, you should know that the semantics of the pointer depends on the type it "points" to. Suppose we wrote `void* val = TypeMap::find(TypeInfo::get<T>);`. Then,
37  * For references (i.e. `T=U&` for some type `U`), `val` is a pointer to the value being referenced; the "true" type of `val` is `T*`.
38  * For small types (i.e. `sizeof(T) <= sizeof(void*)` so this includes all pointers), `val` is literally the value being stored; the "true" type of `val` is `T`
39  * For all other types, `val` is a pointer to the value which is stored on the heap; the "true" type of `val` is `T` and `val` points to a dynamically allocated heap address that the `TypeMap` is responsible for freeing.
40 * `TypeMap::find` crashes when you try to find a type that hasn't been inserted yet while `TypeMap::get` just inserts a new value on the fly.
